1. Introduction to Geospatial Statistics

📌 KEY

Geospatial statistics is the branch of statistics dealing with spatially referenced data. Unlike classical statistics, it considers geographical location and relationships among nearby features.

Spatial statistical methods are widely used in agriculture, environmental monitoring, urban planning, public health, disaster management, and remote sensing. Modern GIS platforms such as ArcGIS Pro and QGIS provide powerful built-in tools for these analyses.

3. Spatial Autocorrelation

📌 DEF

Spatial autocorrelation measures the degree to which nearby locations share similar or dissimilar values. It tests whether observed spatial patterns are clustered, dispersed, or random — and whether they are statistically significant.

Why It Matters:

Most classical statistical methods assume independence between observations — an assumption routinely violated in geographic data. Ignoring spatial autocorrelation leads to biased estimates, incorrect standard errors, and misleading conclusions.

5. Moran's I — Global Spatial Autocorrelation

📌 DEF

Moran's I is the most widely used GLOBAL measure of spatial autocorrelation. It produces a single statistic summarising the overall spatial pattern across the entire study area. Range: approximately −1 (perfect dispersion) to +1 (perfect clustering).

FORMULA | I = (N / S₀) × [ΣΣ wᵢⱼ(xᵢ − x̄)(xⱼ − x̄)] / [Σ(xᵢ − x̄)²]

Variable

Meaning

N

Number of spatial observations

S₀

Sum of all spatial weights (normalization factor)

wᵢⱼ

Spatial weight between locations i and j

xᵢ, xⱼ

Attribute values at locations i and j

x̄

Mean value of the attribute

I

Moran's I statistic (output)

Interpretation Scale

Value Range

Interpretation

Pattern Type

I > 0 (e.g. +0.36)

Positive spatial autocorrelation — similar values cluster together. Closer to +1 = stronger clustering.

CLUSTERED

I ≈ 0

No spatial pattern — values are randomly distributed across the study area.

RANDOM

I < 0 (e.g. −0.45)

Negative spatial autocorrelation — dissimilar values are neighbours. Closer to −1 = stronger dispersion.

DISPERSED

✅ EXAMPLE

Pollution levels across neighbouring industrial zones yield Moran's I = +0.36. This confirms high-pollution areas tend to cluster near other high-pollution zones — warranting targeted remediation.

Applications

Application

Description

Soil nutrient mapping

Identifying clusters of high/low soil fertility across agricultural land

NDVI clustering analysis

Detecting vegetation health patterns from satellite imagery

Crop disease hotspot detection

Finding clusters of disease outbreak in farmland

Rainfall pattern analysis

Understanding precipitation spatial distribution

Urban heat islands

Confirming spatial clustering of high-temperature zones

6. Geary's C — Difference-Based Measure

📌 DEF

Geary's C is a global measure of spatial autocorrelation based on SQUARED DIFFERENCES between neighbouring values — making it more sensitive to local variation and small-scale patterns than Moran's I. Range: 0 to approximately 2.

FORMULA | C = [(N − 1) / 2S₀] × [ΣΣ wᵢⱼ(xᵢ − xⱼ)²] / [Σ(xᵢ − x̄)²]

Variable

Meaning

N

Number of spatial observations

S₀

Sum of all spatial weights

wᵢⱼ

Spatial weight between locations i and j

xᵢ − xⱼ

Squared difference between neighbouring values (key distinction from Moran's I)

x̄

Mean value of the attribute

C

Geary's C statistic (output)

Interpretation Scale

Value Range

Interpretation

Pattern Type

C < 1 (e.g. 0.67)

Positive spatial autocorrelation — neighbouring values are similar. Smaller = stronger clustering.

CLUSTERED

C = 1

No spatial pattern — the distribution is completely random.

RANDOM

C > 1 (e.g. 1.6)

Negative spatial autocorrelation — neighbouring values differ greatly.

DISPERSED

✅ EXAMPLE

Crop yield data across a farming region yields Geary's C = 0.67. Since C < 1, this confirms positive spatial clustering — farms with similar yields are located near each other, likely due to shared soil and irrigation conditions.

Applications

Application

Description

Local variability analysis

Detecting micro-scale spatial differences in field data

Precision agriculture

Identifying within-field soil and crop variability

Soil property analysis

Mapping local soil heterogeneity at fine resolution

Crop stress detection

Finding localised crop stress zones

Ecological studies

Measuring species distribution differences at local scale

8. Getis-Ord Gi* — Hotspot Analysis

📌 DEF

Getis-Ord Gi* is a LOCAL spatial statistic that identifies statistically significant clusters of HIGH values (hotspots) and LOW values (coldspots). Unlike Moran's I, it pinpoints exactly WHERE clusters occur — making it invaluable for applied GIS.

How It Works:

For each feature in the dataset, Gi* evaluates the feature and its neighbours. It produces a z-score and p-value for every location, measuring how the local sum deviates from the global mean.

Gi* Z-Score

Interpretation

Classification

Z ≥ +1.96

Statistically significant cluster of HIGH values

🔴 HOTSPOT (95% confidence)

Z ≥ +2.58

Very strong cluster of HIGH values

🔴🔴 HOTSPOT (99% confidence)

−1.96 < Z < +1.96

No statistically significant spatial pattern

⚪ NOT SIGNIFICANT

Z ≤ −1.96

Statistically significant cluster of LOW values

🔵 COLDSPOT (95% confidence)


Step-by-Step Workflow

1

Input Spatial Data

Attribute values at each location (e.g., crop yield, crime count, NDVI)

2

Define Spatial Neighbourhood

Choose distance band or contiguity rule to define which features are 'neighbours'

3

Calculate Gi* Z-Scores

Standardised deviation of local sum from global mean, accounting for neighbours

4

Calculate P-values

Probability that the observed pattern arose by chance

5

Classify Hotspots / Coldspots

Map significant clusters at 90%, 95%, and 99% confidence levels

9. Kernel Density Estimation (KDE)

📌 DEF

KDE transforms discrete point data into a continuous smooth surface representing the density of events across the study area. Each point contributes to the surface via a kernel function, with influence decreasing with distance — producing a 'heat map' of point concentration.

FORMULA | f̂(x) = (1 / nh) × Σᵢ K((x − xᵢ) / h)

Variable

Meaning

h

Bandwidth (smoothing parameter) — controls the degree of smoothing

n

Total number of points in the dataset

K

Kernel function (typically Gaussian / normal distribution)

x − xᵢ

Distance from evaluation point x to each data point xᵢ

f̂(x)

Estimated density at location x

⚠️ BANDWIDTH

Bandwidth selection is critical: • Too small → noisy, spiky surface — over-fits local variation • Too large → over-smooth surface — obscures local patterns • Optimal: Use Silverman's Rule of Thumb or cross-validation methods in ArcGIS / R

Output Interpretation

Density Area

Meaning

🔴 High-density (HOT)

Many events concentrated in this area

🟡 Medium-density

Moderate concentration of events

🔵 Low-density (COOL)

Few events — sparse distribution

Applications

Application

Description

Crime analysis

Continuous density surface of incidents for patrol planning

Disease mapping

Visualising outbreak spread and intensity zones

Pollution monitoring

Mapping concentration gradients from point measurements

Traffic safety

Locating accident-prone road segments as density peaks

Ecological habitat

Mapping wildlife occurrence density from GPS data

10. Point Pattern Analysis — Core Concepts

📌 DEF

Point Pattern Analysis (PPA) studies the spatial arrangement of discrete point events — disease cases, crime locations, tree positions, well locations — to determine whether their distribution is Clustered, Random, or Regular.

Key Principle: PPA focuses on GEOMETRIC ARRANGEMENT rather than attribute values. Understanding the pattern reveals the underlying process that generated it.

CLUSTERED

RANDOM (CSR)

REGULAR

Points occur close together in groups — indicating attraction or a common underlying cause.

Points occur without any systematic order. Each location equally likely — the null hypothesis.

Points are evenly spaced — indicating repulsion, competition, or deliberate planning.

Example: Disease cases from contaminated water source; crop pest outbreaks

Example: Raindrops on a flat surface; trees in undisturbed uniform forest

Example: Orchard tree rows; fire stations for equal city coverage

💡 NOTE

Point patterns are influenced by both FIRST-ORDER effects (variation in intensity across the study area — e.g., population density) and SECOND-ORDER effects (interactions between points — e.g., contagion). Distinguishing these is essential for correct interpretation.

11. Nearest Neighbour Index (NNI)

📌 DEF

The NNI compares the OBSERVED mean distance between each point and its nearest neighbour to the EXPECTED mean distance under Complete Spatial Randomness (CSR). The ratio reveals whether points are clustered, random, or regularly spaced.

FORMULA | NNI = d_obs / d_exp where d_exp = 1 / (2√λ) and λ = n / A

Variable

Meaning

d_obs

Observed mean nearest-neighbour distance (calculated from your data)

d_exp

Expected mean distance under Complete Spatial Randomness

λ

Point density (number of points n divided by study area A)

NNI

Nearest Neighbour Index — the ratio of observed to expected distance

Interpretation Scale (Range: 0 → 2.15)

Value Range

Interpretation

Pattern Type

NNI = 0.0

Maximum clustering — all points at the same location

MAXIMUM CLUSTER

NNI < 1 (e.g. 0.56)

Observed distances < expected. Points closer than random.

CLUSTERED

NNI = 1

Observed distances match expected random distribution.

RANDOM (CSR)

NNI > 1 (e.g. 1.75)

Observed distances > expected. Points more evenly spaced.

REGULAR

NNI = 2.15

Maximum regularity — perfect hexagonal lattice

MAXIMUM REGULAR

📐 SIGNIFICANCE TEST

Statistical significance is tested via a z-score: z = (NNI − 1) / SE. A z-score beyond ±1.96 (95% confidence) indicates the pattern is unlikely by chance. ArcGIS, QGIS, and R all compute this automatically.

✅ WORKED EXAMPLE

Retail store study: observed mean nearest-neighbour distance = 0.312 km; expected under CSR = 0.559 km. NNI = 0.312 / 0.559 = 0.558. Well below 1 → stores are significantly clustered, likely near high-traffic corridors.

Limitations of NNI

Limitation

Explanation

Single-scale only

Only considers the NEAREST neighbour — misses clustering at broader distances

Edge effects

Irregular boundaries can bias results if edge corrections are not applied

Homogeneity assumption

Assumes uniform study area; fails with population or density heterogeneity

12. Ripley's K Function — Multi-Scale Analysis

📌 DEF

Ripley's K is a MULTI-SCALE point pattern analysis tool that examines clustering, randomness, or dispersion across a RANGE of distances — not just at a single scale like NNI. It reveals how spatial structure changes with distance.

FORMULA | K(r) = (A / n²) × ΣᵢΣⱼ wᵢⱼ · I(dᵢⱼ ≤ r) → L(r) = √(K(r) / π) − r

Variable

Meaning

A

Total area of the study region

n

Total number of points

wᵢⱼ

Edge correction weight for pair i, j

I(dᵢⱼ ≤ r)

Indicator: 1 if distance between i and j ≤ r, else 0

r

Search distance (radius) — varied across a range

K(r)

Ripley's K statistic at distance r

L(r)

Besag's L transformation — linearises K(r) so L=0 means CSR

Interpreting the L(r) Plot

Value Range

Interpretation

Pattern Type

L(r) > 0

Clustering at distance r — more points within r than expected under CSR

CLUSTERED at this scale

L(r) = 0

Random pattern at distance r — matches CSR expectation

RANDOM at this scale

L(r) < 0

Dispersion at distance r — fewer points within r than expected

DISPERSED at this scale

✅ KEY EXAMPLE

Trees in a forest: clustered at small scales (seed dispersal → juveniles grow near parent) but regular at large scales (competition for light and water → self-thinning). NNI would give an ambiguous single value; Ripley's K reveals BOTH patterns.

💡 CSR ENVELOPE

Software generates a simulation envelope from multiple random patterns. If the observed K/L curve lies ABOVE the upper envelope → significant clustering. BELOW the lower envelope → significant dispersion. Between the envelopes → random pattern at that scale.

13. Spatial Interpolation

📌 DEF

Spatial interpolation estimates values at UNKNOWN locations using known sample points. Underpinned by Tobler's Law — nearby points are more similar than distant ones — it converts point observations into continuous spatial surfaces.

Category

Method

Description

Deterministic

IDW

Weights decrease with distance from known points — simple and fast

Deterministic

Spline

Smooth surface fitted through known points — good for gradual changes

Geostatistical

Kriging

Optimal prediction using spatial autocorrelation; provides uncertainty estimates

FORMULA | IDW: Z(x₀) = Σ [Zᵢ / dᵢᵖ] / Σ [1 / dᵢᵖ]

Variable

Meaning

Z(x₀)

Estimated value at unknown location x₀

Zᵢ

Known sample value at point i

dᵢ

Distance from sample point i to the unknown location

p

Power parameter (typically 2) — controls the rate of distance decay

Method Comparison

Method

Advantages

Disadvantages

Best Used For

IDW

Simple, fast, intuitive

No uncertainty; edge artifacts

Dense samples, quick estimates

Spline

Smooth, good for gradients

Overshoots with outliers

Elevation, temperature maps

Kriging

Optimal; uncertainty estimates

Complex; needs variogram

Precision agriculture, minerals

⭐ KRIGING

Kriging is considered the gold standard because it INCORPORATES SPATIAL AUTOCORRELATION via a variogram model — directly linking it to Moran's I and Geary's C concepts. It also provides prediction uncertainty (standard error) at each interpolated point.
