## 🔄 CONTINUOUS HOMEOSTASIS
## The Body Never Stops Running

Just like a human body maintains temperature, blood pressure, and breathing
even while sleeping, this system runs continuous background processes:

### Always-On Autonomic Functions (Brain Stem)

1. **Heartbeat** → `graphify update .` runs after every code change to keep
   the knowledge graph current and dependency maps alive.

2. **Breathing** → `lint-and-validate` runs on every file save to inhale
   clean code and exhale errors.

3. **Immune Patrol** → `vulnerability-scanner` continuously watches for
   security threats in dependencies.

4. **Memory Consolidation** → `memory-system` and `continuous-learning`
   store session insights for future recall.

5. **Waste Removal** → `simplify-code` and `clean-code` periodically
   identify and remove dead code, unused imports, and stale dependencies.

### Expert-to-Expert Relay (Synaptic Transmission)

When one organ encounters something outside its specialty, it passes
the signal to the next expert organ:

```
Frontend Specialist encounters a database bottleneck:
  → Signals Backend Specialist (muscle relay)
  → Backend Specialist detects query issue
  → Signals Database Architect (organ consultation)
  → Database Architect optimizes schema
  → Signals Performance Optimizer (cerebellum refinement)
  → Performance Optimizer validates benchmarks
  → Signals QA Engineer (taste test)
  → QA confirms improvement
  → Result flows back to Frontend Specialist
```

## 🏥 HOW TO ACTIVATE THE BODY

### Activation Protocol (For Every Task)

1. **Stimulus arrives** → User sends a request
2. **Ears parse** → Socratic Gate asks clarifying questions if needed
3. **Eyes scan** → Explorer checks codebase state, reads relevant files
4. **Brain routes** → Orchestrator identifies which organs to activate
5. **Organs execute** → Specialist agents work in parallel (arms build, skeleton structures, immune defends)
6. **Heart pumps** → Graphify keeps the knowledge graph flowing
7. **Tongue tastes** → Tests validate the output
8. **Mouth speaks** → Clean, documented result delivered
9. **Memory stores** → Learnings saved for next session
10. **Lungs breathe** → Resources cleaned up, dead code removed

### The Golden Rule

> **No organ works alone. Every output passes through at least TWO organs
> before reaching the user. This is how excellence compounds.**
